# 13. CLI 安装、登录与第一次改代码

CLI 适合已经习惯终端的人。它的优势是直接进入项目目录，让 Codex 读取当前仓库、改文件、跑检查。

## 基本流程

```bash
cd /path/to/project
codex
```

进入后，先让 Codex 做只读检查：

```text
请阅读这个仓库，找出入口文件、测试命令和构建命令。暂时不要改文件。
```

## 第一次改代码

推荐从小任务开始：

```text
请把 README 里的安装步骤整理得更清楚。
约束：保留所有原链接，不新增外部依赖。
完成后告诉我改了哪些文件。
```

## 常用命令

```bash
git status --short
rg "关键词"
python3 -m http.server 8765
```

这些命令分别用于看改动、搜索内容、本地预览静态站。

## CLI 使用原则

- 先读项目，再改文件。
- 先跑最快检查，再跑完整检查。
- 不懂命令时让 Codex 解释用途。
- 涉及删除、重置、清理前先确认。

## 完成标准

- 你能在项目目录启动 Codex。
- 你能让它只读分析项目。
- 你能让它完成一个小改动。
- 你能用 `git diff` 或 Codex 的变更视图检查结果。
