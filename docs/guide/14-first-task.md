# 14. 第一个可验证任务: README 变网页

第一个任务建议做“把 README 变成一个能打开的网页”。它能练到 Codex 最重要的闭环：读文件、改文件、预览、检查、修复。

## 任务目标

把 README 的核心内容整理成一个静态网页，保留原来的中转站、购买和教程链接。

## 推荐提示词

```text
请把 README.md 的核心内容整理成一个静态首页。
要求：
1. 保留原来的国内站、备用入口、购买套餐、飞书教程链接。
2. 首页不要太长，只保留入口、学习路线和实战案例入口。
3. 长教程放到 docs/guide，案例放到 recipes。
4. 完成后用本地预览检查页面是否能打开。
```

## Codex 应该做什么

- 读取 README 和现有首页。
- 找出现有链接。
- 设计页面结构。
- 修改 HTML 或 Markdown。
- 启动本地预览。
- 检查链接是否保留。
- 汇报改动。

## 验证命令

```bash
python3 -m http.server 8765
```

然后打开：

```text
http://127.0.0.1:8765/
```

## 完成标准

- 首页能打开。
- 原来的国内站和中转链接还在。
- `docs/guide` 和 `recipes` 有入口。
- Codex 汇报了改动文件。
- 你能从首页进入第一个实战案例。
