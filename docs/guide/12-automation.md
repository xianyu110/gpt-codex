# 12. 定时任务 / 自动化入门

自动化适合稳定、重复、可验证的任务。不要把还没跑通的复杂流程一上来就自动化。

## 适合自动化的任务

- 每天检查某个网页是否有新内容。
- 定时生成日报或周报。
- 监控 GitHub Actions 是否失败。
- 定时抓取公开数据并入库。
- 到点提醒你处理某个 Thread。

## 不适合自动化的任务

- 需求还在变化的任务。
- 每次都需要人工判断的任务。
- 会处理敏感账号或支付信息的任务。
- 失败后没有日志、无法定位原因的任务。

## 自动化设计清单

| 项目 | 要求 |
| --- | --- |
| 触发条件 | 每天几点、每小时、还是事件触发 |
| 输入 | URL、仓库、文件夹、关键词 |
| 去重 | 如何避免重复处理 |
| 输出 | Markdown、消息、PR、表格 |
| 日志 | 成功和失败记录在哪里 |
| 人工确认 | 哪些步骤不能自动执行 |

## 推荐提示词

```text
请把这个已经跑通的流程设计成 Codex 自动化。
要求：只处理新增内容；保留 state.json 去重；失败写入 logs/errors.jsonl；
每次汇报扫描数量、新增数量、成功数量、失败数量和日志路径。
```

## 完成标准

一个自动化至少要有：

- 明确触发频率。
- 去重状态。
- 日志路径。
- 失败处理。
- 汇报格式。
