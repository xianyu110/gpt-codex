# 00. 学习路线: 这份教程怎么用

Codex 不是只会回答问题的聊天机器人，而是一个能围绕文件、命令、浏览器、外部工具和任务目标持续推进的 coding agent。小白学习 Codex，最容易卡在两个地方：一是入口太多，不知道先装什么；二是能用了以后，不知道应该让它做什么。

这份教程按“先跑通，再做事，再沉淀”的顺序设计。

## 你要先理解的三件事

1. Codex 的核心不是“写代码”，而是“执行任务”。
2. 一条 Thread 就是一件事，任务越清楚，结果越稳定。
3. AGENTS.md、Skills、MCP、自动化，是把临时经验变成长期工作流的关键。

## 推荐学习顺序

| 阶段 | 目标 | 对应章节 |
| --- | --- | --- |
| 跑起来 | 完成安装、登录、订阅或中转配置 | 01-03 |
| 看懂界面 | 知道项目、线程、权限、按钮怎么用 | 04-06 |
| 会下任务 | 能用 Plan Mode、斜杠命令和推理深度 | 07-10 |
| 变成工作流 | 用 Skills、MCP、自动化做可复用流程 | 11-12 |
| 实战验证 | 做一个能打开、能检查、能交付的小项目 | 13-14 + recipes |

## 每章怎么读

每章都按同一套结构写：

- 适合谁
- 你要准备什么
- 具体怎么做
- 常见坑
- 完成标准

不要一开始就追求“大而全”。第一目标是让 Codex 成功改一个文件、跑一个检查、给你一个可以验证的结果。

## 新手最推荐的第一条任务

把一个 README 改成静态网页。

这个任务足够小，但能练到 Codex 最关键的能力：读文件、改文件、遵守风格、启动预览、截图检查、修复细节。

直接看：[第一个可验证任务](14-first-task.md)。
