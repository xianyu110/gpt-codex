# 09. 推理深度: High / Extra High 怎么选

推理深度影响 Codex 思考时间、成本和稳定性。不是所有任务都需要最高。

## 快速选择表

| 任务 | 推荐推理深度 |
| --- | --- |
| 改文案、改链接、简单 Markdown | Low / Medium |
| 小型代码修改、补测试 | Medium |
| 多文件重构、复杂 bug | High |
| 架构设计、长链路调试、大型迁移 | Extra High |

## 新手建议

默认 Medium。遇到下面情况再升 High：

- Codex 连续改错。
- 它没理解项目结构。
- 需要跨多个文件判断因果。
- 需要先做方案权衡。

Extra High 适合“少量高价值任务”，比如：

- 真实仓库疑难 bug。
- 复杂迁移。
- 长时间自动化。
- 安全风险较高的改动。

## 不要用高推理掩盖坏提示词

坏提示词：

```text
帮我弄好。
```

好提示词：

```text
目标：把 docs/guide 补齐为 15 篇入门教程。
约束：保留原有中转链接，不改购买入口，不新增统计代码。
完成标准：每篇至少包含目的、步骤、注意事项和完成标准。
```
