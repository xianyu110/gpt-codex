# 04. 桌面 App 界面总览

Codex 桌面 App 的重点不是“聊天窗口”，而是围绕项目和 Thread 组织工作。

## 常见区域

| 区域 | 作用 | 新手建议 |
| --- | --- | --- |
| Thread 列表 | 每条任务的上下文 | 一个任务开一条 Thread |
| 输入框 | 写目标、约束、补充信息 | 写清 Goal / Context / Done when |
| Plan Mode | 先规划再执行 | 复杂任务先打开 |
| 文件/工作区 | Codex 能读取和修改的项目 | 只添加当前任务需要的目录 |
| 工具输出 | 命令、测试、浏览器结果 | 出错时先看这里 |
| Diff / Changes | 查看文件改动 | 提交前必须过一遍 |

## 第一次打开后做什么

1. 添加项目文件夹。
2. 新建 Thread。
3. 输入：“请阅读这个仓库，告诉我主要文件和运行方式，暂时不要改文件。”
4. 看它是否能正确识别 README、配置文件、入口文件。
5. 再给一个小修改任务。

## 不要把 Codex 当搜索框

好的任务不是：

```text
帮我优化一下。
```

更好的任务是：

```text
请把 README 里的安装步骤整理成 docs/guide/01-install.md，
保留现有链接，不要改首页。完成后列出改了哪些文件。
```

## 完成标准

- 你知道如何新建 Thread。
- 你知道如何查看 Codex 改了哪些文件。
- 你知道如何让它先规划再执行。
- 你知道每次任务结束后要看 diff 和验证结果。
