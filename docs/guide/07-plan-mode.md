# 07. Plan Mode: 先规划再开发

Plan Mode 适合复杂、模糊、有风险的任务。它让 Codex 先读上下文、问问题、列计划，再进入执行。

## 什么时候开 Plan Mode

- 不确定要改哪些文件。
- 任务影响多个模块。
- 需要先比较方案。
- 你想先确认风险。
- 你在让 Codex 改真实业务仓库。

## 什么时候不用

- 改一个错别字。
- 补一个小链接。
- 跑一个简单命令。
- 你已经明确指定文件和改法。

## 好的 Plan 请求

```text
请先进入 Plan Mode。目标是把这个仓库改造成 Codex 中文教程站。
先检查现有文件结构和链接，尤其不要删除原来的中转站链接。
然后给出 3-6 条实施计划，等我确认后再改文件。
```

## Plan 看什么

你要重点看：

- 是否识别了正确文件。
- 是否保留了安全和链接约束。
- 是否有验证步骤。
- 是否把范围扩大得太多。

## 完成标准

一份好计划至少包括：

- 要改哪些文件。
- 为什么改这些文件。
- 不改哪些东西。
- 怎么验证。
- 风险是什么。
