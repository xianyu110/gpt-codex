# 11. Skills 与 MCP 入门

Skills 和 MCP 是 Codex 从“会干一次”变成“以后都能复用”的关键。

## Skills 是什么

Skill 是一套本地可复用说明，可以包含：

- SKILL.md
- 模板
- 脚本
- 示例
- 检查清单

适合沉淀成 Skill 的任务：

- 公众号文章整理。
- PPT 生成。
- PDF 审阅。
- 飞书多维表格操作。
- 特定仓库发布流程。

## MCP 是什么

MCP 是让 Codex 连接外部工具或数据源的协议。它适合处理“Codex 自己看不到，但工具能拿到”的东西，比如：

- 浏览器自动化。
- GitHub issue / PR。
- Notion / 飞书 / Slack。
- 数据库。
- 内部业务系统。

## 怎么选

| 需求 | 用什么 |
| --- | --- |
| 一套重复写作流程 | Skill |
| 需要访问外部系统 | MCP |
| 需要定时执行 | Automation |
| 只想本次任务说明清楚 | 当前提示词 |
| 仓库长期规则 | AGENTS.md |

## 第一个 Skill 应该怎么写

先不要追求复杂。一个好的 SKILL.md 至少写：

- 什么时候使用。
- 输入需要什么。
- 输出是什么。
- 禁止做什么。
- 验证标准。

## 安全提醒

MCP 能连接真实外部系统，配置前要确认：

- 权限范围。
- 是否会写入数据。
- 是否会发送消息。
- 是否会读取隐私数据。
- 是否需要人工确认。
