1. 掌握Codex的基本操作与配置

## 第三章 项目组织与管理

### 3.1 学术研究项目结构建议

建议采用以下文件夹结构组织您的研究项目：

```
research/
├── data/                    # 数据文件目录
│   ├── raw/                 # 原始数据（只读）
│   ├── processed/           # 处理后数据
│   └── external/            # 外部数据源
├── analysis/                # 分析脚本目录
│   ├── preprocessing/       # 数据预处理脚本
│   ├── statistics/          # 统计分析脚本
│   └── visualization/       # 可视化脚本
├── papers/                  # 论文与报告
│   ├── drafts/              # 草稿
│   └── figures/             # 图表导出
├── references/              # 参考文献
└── README.md                # 项目说明文档
```

### 3.2 Threads系统在研究中的应用

Codex采用双层管理结构：

| 层级 | 名称 | 学术应用场景 |
|------|------|--------------|
| 第一层 | 工作区（文件夹） | 对应一个研究课题或项目 |
| 第二层 | Thread（线程） | 对应具体分析任务 |

**建议的组织原则**：
- 每个研究课题创建独立工作区
- 同一工作区内的Thread按分析任务划分
- 避免在单一Thread中混用不同分析任务

**Threads系统的设计优势**：

| 传统方式的痛点 | Codex的解决方案 |
|----------------|-----------------|
| 上午写网页、下午算Excel、晚上改文案，所有东西搅成一锅粥 | 不同任务分到不同Thread |
| 上下文污染严重 | 对话和目标互不污染 |
| AI开始胡编乱造 | 随时可以断点续写 |
| 找不到文件在哪 | 文件和资源井井有条 |

### 3.3 添加研究项目

1. 在本地文件系统创建项目文件夹
2. 在Codex中点击"添加文件夹"
3. 选择对应的研究项目目录
4. 创建新的Thread开始工作


## 第五章 数据处理实践

### 5.1 数据清洗工作流

**场景**：清洗实验数据，处理缺失值和异常值

**操作步骤**：

1. 创建新Thread，命名为"数据清洗"
2. 进入Plan模式（输入`/plan`）
3. 描述数据清洗需求

**示例提示词**：

```
我有一份实验数据文件位于 data/raw/experiment_results.csv，
包含以下列：subject_id, group, condition, reaction_time, accuracy。

请帮我完成以下数据清洗任务：
1. 检查并报告缺失值分布
2. 识别reaction_time列中的异常值（±3SD）
3. 对accuracy列进行逻辑检查（0-1范围）
4. 生成清洗后的数据文件和清洗报告
```

4. 审核生成的计划
5. 确认执行

### 5.2 统计分析工作流

**场景**：进行组间差异检验

**示例提示词**：

```
对清洗后的数据 data/processed/cleaned_data.csv 进行以下分析：

1. 描述性统计：按group和condition分组计算reaction_time的均值、标准差
2. 正态性检验：Shapiro-Wilk检验
3. 方差齐性检验：Levene检验
4. 双因素方差分析：group × condition
5. 事后检验：Tukey HSD
6. 效应量计算：偏η²

请生成完整的分析代码和结果报告。
```

### 5.3 可视化工作流

**场景**：生成论文级图表

**示例提示词**：

```
基于上述分析结果，生成以下图表：

1. 箱线图：展示各组reaction_time分布，按condition分面
2. 交互作用图：展示group × condition的均值和误差条

要求：
- 使用ggplot2风格
- 配色方案：colorblind-friendly
- 图像尺寸：8×6英寸，300 DPI
- 保存至 papers/figures/ 目录
```

## 第六章 高级功能应用

### 6.1 定时任务：自动化数据采集

对于需要定期采集数据的研究项目，可配置定时任务：

**配置步骤**：

1. 进入定时任务设置界面
2. 创建新任务
3. 配置执行时间和频率

**示例配置**：

| 配置项 | 设置 |
|--------|------|
| 任务名称 | 每日数据同步 |
| 执行频率 | 每日 09:00 |
| 任务内容 | 从数据源同步最新数据至本地 |

**实际应用案例**：

| 场景 | 说明 |
|------|------|
| 服务器托管 | 项目跑在云服务器上 |
| 自动化巡检 | Codex每天早上9点自动对服务器进行巡检，检查报错 |
| 自动修复 | 如果有报错，会自己解决，然后总结原因 |
| 通知推送 | 通过飞书机器人发送通知 |

### 6.2 Skills：自定义分析流程

对于重复性分析任务，可创建自定义Skill：

**创建步骤**：

1. 点击"New Skill"
2. 描述Skill功能
3. 测试并保存

**示例：论文图表生成器Skill**

```
该Skill用于根据数据分析结果自动生成符合期刊投稿要求的图表：
- 自动设置合适的图像尺寸和分辨率
- 应用色盲友好的配色方案
- 生成符合APA/IEEE格式的图例和标注
- 导出多种格式（PNG, PDF, SVG）
```

**Codex的Skills优势**：

| 对比项 | Codex | Claude Code / OpenCode |
|--------|-------|------------------------|
| Skills管理 | 图形化界面，轻松管理 | 不知道装了哪些、装到哪、有什么用 |
| Skills创建 | 内置Skill Creator | 需要自己找、安装 |

## 第七章 质量控制与验证

### 7.1 代码验证原则

| 验证项目 | 方法 | 重要性 |
|----------|------|--------|
| 逻辑正确性 | 小数据集测试 | ★★★★★ |
| 结果可复现 | 设置随机种子 | ★★★★★ |
| 边界条件 | 极端值测试 | ★★★★ |
| 性能 | 大数据集测试 | ★★★ |

### 7.2 常见问题与解决

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 结果不可复现 | 未设置随机种子 | 在代码开头添加seed设置 |
| 统计结果异常 | 数据分布假设不满足 | 检验假设条件，考虑非参数方法 |
| 图表显示错误 | 数据格式问题 | 检查数据类型转换 |
| 运行超时 | 数据量过大 | 分块处理或优化算法 |

### 7.3 最佳实践清单

- [ ] 始终保留原始数据备份
- [ ] 记录所有数据处理步骤
- [ ] 使用版本控制管理代码
- [ ] 对关键结果进行交叉验证
- [ ] 定期导出和备份分析结果
