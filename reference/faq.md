# FAQ

## 我不会写代码，能用 Codex 吗？

能。你可以先把 Codex 当成“会读文件、改文件、跑检查的执行型助手”。从 README 变网页、整理 Obsidian、生成 PPT 这种任务开始。

## 我应该先用 App 还是 CLI？

小白先用桌面 App，开发者可以直接用 CLI。App 更容易看到 Thread、计划和改动；CLI 更适合真实仓库。

## Plan Mode 必须开吗？

不是。小任务不用开。复杂、多文件、有风险的任务建议先开。

## Full access 要不要一直开？

不要。默认 Workspace write。只有跨目录、系统配置、部署或本机自动化时才考虑更高权限。

## 第三方中转是不是官方订阅？

不是。第三方中转的价值是快速可用、低门槛和配置省心；官方订阅是 OpenAI 官方账号套餐。两者要分清楚。

## 原来的购买和中转链接还保留吗？

保留。教程改造的原则是内容变完整，原来的国内站、备用入口、套餐购买、中转 API、飞书教程链接不丢。

## 一条 Thread 可以做很多事吗？

不建议。一条 Thread 做一件事，结果更稳，也更容易 review。

## Codex 改完就算完成了吗？

不算。至少要检查：

- 改了哪些文件。
- 是否符合要求。
- 是否保留关键链接。
- 是否跑过相关验证。
- 是否有剩余风险。
