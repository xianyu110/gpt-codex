# Codex x 真实仓库: 修 bug + 补测试

## 适合谁

适合开发者用 Codex 处理真实代码仓库里的 bug。

## 准备输入

- bug 描述。
- 复现步骤。
- 错误日志。
- 期望行为。

## 提示词

```text
请修复这个 bug。
上下文：报错日志如下，复现步骤如下。
要求：先定位根因；只改必要文件；补或更新测试；先跑最快相关测试。
```

## Codex 应该做什么

- 复现问题。
- 定位根因。
- 做最小修复。
- 补测试。
- 跑验证。

## 验证

- bug 不再复现。
- 相关测试通过。
- diff 范围合理。

## 风险

- 没复现就猜修。
- 顺手重构太多。
- 测试没有覆盖真实问题。
